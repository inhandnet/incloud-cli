# CLI 场景化验收测试设计

## 目标

开发完 CLI 命令后，让 AI 模拟真实用户在真实场景中使用命令，验证命令能否满足实际运维需求。发现问题时，输出对命令设计的具体改进建议。

## 两级模式

### 快速模式（implement-module 自动触发）

- **触发时机**：implement-module Phase 3 集成验证后自动调用
- **范围**：仅测试本次新增/修改的命令
- **深度**：命令可用性 + 输出格式检查
- **时间预期**：1-2 个场景，快速跑完

### 完整模式（`/test-scenarios` 手动触发）

- **触发时机**：手动调用 `/test-scenarios`
- **范围**：推断所有相关场景，完整验收
- **深度**：模拟真实用户端到端解决问题，包含语义验证和设计反馈
- **时间预期**：按场景数量，可并行多个 agent

## 实现组件

### 1. `/test-scenarios` skill

**职责**：场景推断 + agent 调度 + 报告汇总

**输入**：
- 可选参数：指定场景名（如 `diagnostics`、`fleet-inspection`）
- 无参数时：自动推断（基于 git diff 找出变更的命令，匹配 SKILL.md + references 中的场景）

**流程**：
1. 识别待测命令（从参数或 git diff）
2. 读取 incloud-skills 的 SKILL.md + references 目录
3. 推断命令涉及哪些运维场景
4. 为每个场景启动 network-engineer agent（可并行）
5. 收集各 agent 的验收报告
6. 汇总输出最终报告

**输出**：结构化验收报告（见下方格式）

### 2. network-engineer agent 升级

**新增要求**：在现有基础上增加结构化输出

agent 在跑场景时，除了正常的运维操作，还要关注并记录：

**命令可用性**：
- 命令是否存在且能执行
- 参数是否符合直觉（网工第一反应会怎么用）
- 错误信息是否足够定位问题

**输出质量**：
- table 输出是否包含排查所需的关键字段
- JSON/YAML 输出结构是否便于管道处理
- 字段命名是否清晰

**流程连贯性**：
- 从 A 命令的输出能否自然衔接到 B 命令的输入（如 list 拿 ID → get 查详情）
- 排查链路是否有断裂点（某个环节需要的信息 CLI 拿不到）

**设计建议**：
- 缺少什么命令或 flag 会让场景更顺畅
- 现有命令的哪些设计可以优化
- 输出格式的改进建议

### 3. 验收报告格式

```markdown
# 场景验收报告：<场景名>

## 场景描述
<一句话描述模拟的场景>

## 测试环境
- context: <dev/demo>
- 涉及设备: <SN 或 ID>

## 执行过程
### Step 1: <操作描述>
- 命令: `incloud xxx`
- 结果: PASS / FAIL / WARN
- 输出摘要: <关键数据>
- 备注: <如有问题，描述问题>

### Step 2: ...

## 总结

### 通过项
- <命令1>: 可用，输出符合预期

### 问题项
- <命令2>: <具体问题描述>

### 设计建议
- [ ] <建议1：具体的改进项，附理由>
- [ ] <建议2>
```

### 4. implement-module 集成

在 implement-module 的 Phase 3（集成验证）之后，新增一步：

```
Phase 3.5: 场景快速验证
- 读取本次实现的命令列表
- 调用 test-scenarios 快速模式
- 将验收结论作为 Phase 4 cleanup 的输入
  - PASS: 正常进入 cleanup
  - WARN/FAIL: 展示问题，由开发者决定是否修复后再继续
```

## 不做的事

- **不做 mock 场景测试**：场景测试的价值就在于打真实 API，mock 没有意义
- **不做自动修复**：agent 只报告问题和建议，修不修、怎么修由开发者决定
- **不做回归基线**：不保存历史结果做对比，每次独立验收（场景本身会变，基线维护成本高）
- **不替代单元测试**：单元测试验证 API 调用正确性，场景测试验证用户体验，两层互补

## 实现顺序

1. 升级 network-engineer agent（增加结构化输出要求）
2. 创建 `/test-scenarios` skill
3. 修改 implement-module skill 集成快速模式
